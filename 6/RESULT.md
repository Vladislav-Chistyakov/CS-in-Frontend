# 🧪 Исследование производительности операций с массивами в JavaScript

## 🎯 Цель

Изучить, как операции с массивами в JavaScript влияют на производительность, а также сравнить:

* `push` / `pop`
* `shift` / `unshift`
* массивы с дырками (holey) и без дырок (packed)

---

## ⚙️ Методика

Для тестирования использовались массивы разного размера:

```
10, 100, 1 000, 10 000, 100 000
```

Для каждого размера проводились замеры:

* массив с дырками: `new Array(size)`
* массив без дырок: `new Array(size).fill(0)`

Каждый тест:

1. Прогревался (JIT warm-up)
2. Выполнялся 10 раз
3. Считалось среднее время выполнения

---

## 🔬 Что измерялось

Для каждого метода выполнялся следующий сценарий:

* `push` — добавление элементов в конец массива
* `pop` — удаление элементов с конца массива
* `shift` — удаление элементов из начала массива
* `unshift` — добавление элементов в начало массива

Операции выполнялись `size` раз.

---

## 📊 Результаты

### Быстрые операции

* `push`
* `pop`

Практически не зависят от размера массива.

➡️ Сложность: **O(1)**

---

### Медленные операции

* `shift`
* `unshift`

Время выполнения резко растёт с увеличением размера массива.

➡️ Сложность: **O(n)**

---

### Особенности `unshift`

Операция `unshift` показала наихудшую производительность:

* при каждом вызове происходит сдвиг всех элементов массива
* при многократном выполнении приводит к **квадратичному росту времени**

➡️ Фактически: **O(n²)** в данном сценарии

---

### Holey vs Packed

* массивы с дырками (holey) показали немного худшую производительность
* разница незначительна в рамках данного эксперимента

---

## 📌 Выводы

* `push` и `pop` — самые эффективные операции для работы с массивом
* `shift` и `unshift` — дорогие операции, особенно на больших массивах
* `unshift` — наиболее затратная операция
* массивы с дырками могут снижать производительность

---

## 🧠 Рекомендации

* использовать `push` / `pop` для работы с массивами
* избегать `shift` / `unshift` на больших данных
* по возможности не создавать массивы с дырками

---

## 🚀 Итог

Операции с массивами в JavaScript сильно отличаются по производительности в зависимости от типа операции и структуры массива. Понимание этих различий позволяет писать более эффективный код.
